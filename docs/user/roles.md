# Роли и права доступа

В системе 5 ролей, каждая с определённым набором прав. Роли иерархичны — более высокая роль включает права нижестоящих.

## Суперадмин (`superadmin`)

Роль платформы. Управляет всеми заведениями HookahPOS.

**Доступ:**
- Список всех заведений
- Управление подписками (смена плана, продление)
- Имперсонация — вход в любое заведение от имени владельца
- Управление тарифными планами
- Создание и просмотр сетей заведений

**Ограничения:**
- Не привязан к конкретному заведению
- Вход только по email/паролю на главном домене

## Владелец сети (`chain_owner`)

Управляет сетью из нескольких заведений.

**Доступ:**
- Список заведений сети с мини-KPI
- Создание новых заведений в сети
- Поиск и добавление существующих заведений
- Имперсонация — вход в любое заведение сети от имени владельца
- Агрегированные отчёты по сети:
  - Дашборд сети
  - Продажи с разбивкой по заведениям
  - Сравнение заведений
  - Топ товаров по сети

**Как стать:**
- Суперадмин создаёт сеть и назначает chain_owner
- Владелец заведения (с планом Business) создаёт сеть через раздел «Сеть»

## Владелец (`owner`)

Полный контроль над своим заведением. Создаётся автоматически при регистрации.

**Полный доступ, включая:**
- Все функции администратора (см. ниже)
- Настройки заведения (название, логотип, акцентный цвет)
- Настройки чеков (ширина, шапка, подвал)
- Управление подпиской (смена плана)
- Настройка интеграций (ЕГАИС, Честный знак)
- Приглашение сотрудников по email

**Вход:**
- Email/пароль на главном домене (с редиректом на поддомен)
- Email/пароль на поддомене
- PIN-код (если настроен)

## Администратор (`admin`)

Управление операционной деятельностью заведения.

**Доступ:**

| Раздел | Операции |
|--------|---------|
| **Товары** | Создание, редактирование, удаление, техкарты |
| **Категории** | Создание, редактирование, удаление |
| **Ингредиенты** | Создание, редактирование, удаление |
| **Группы ингредиентов** | Создание, редактирование, удаление |
| **Цеха** | Создание, редактирование, удаление |
| **Поставки** | Приёмка товара |
| **Инвентаризация** | Создание, ввод, применение |
| **Сотрудники** | Создание, редактирование, деактивация |
| **Гости** | Создание, редактирование, удаление |
| **Залы и столы** | Создание, удаление столов |
| **Отчёты** | Все отчёты (продажи, товары, сотрудники, скидки) |
| **Складские остатки** | Отчёт по остаткам |
| **Касса** | Открытие/закрытие дня, работа с заказами |
| **ЕГАИС** | Входящие ТТН, перемещение, списание |
| **Маркировка** | Списание маркированных единиц |

**Недоступно:**
- Настройки заведения (брендинг)
- Настройки чеков
- Управление подпиской
- Настройка интеграций

**Вход:**
- Email/пароль на поддомене
- PIN-код (если настроен)

## Кассир (`cashier`)

Работа с кассой — создание и оплата заказов.

**Доступ:**

| Раздел | Операции |
|--------|---------|
| **Касса** | Открытие/закрытие кассового дня |
| **Заказы** | Создание, добавление позиций, оплата, отмена |
| **Карта зала** | Просмотр столов, выбор стола |
| **Каталог** | Просмотр товаров (только чтение) |
| **Гости** | Поиск и выбор гостя при оплате (только чтение) |
| **Маркировка** | Сканирование кодов при продаже |

**Недоступно:**
- Создание/редактирование товаров, категорий
- Управление сотрудниками
- Поставки, инвентаризация
- Отчёты и статистика
- Настройки

**Вход:**
- PIN-код на поддомене (основной способ)
- Email/пароль (если настроен)

## Имперсонация

Имперсонация позволяет войти в заведение «от имени владельца» без знания его пароля:

- **Суперадмин** может имперсонировать любое заведение (токен на 8 часов)
- **Владелец сети** может имперсонировать заведения своей сети (токен на 8 часов)

При имперсонации пользователь получает все права **владельца** выбранного заведения.
